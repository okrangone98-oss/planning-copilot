공고문이 원하는 답을 먼저 읽습니다

지원사업의 목적, 평가기준, 금지사항, 가점 요소를 분해하면 같은 아이디어도 훨씬 정확한 언어로 기획됩니다.

공고문 자동 분석 공고문을 붙여넣거나 텍스트 파일을 첨부한 뒤 누르면 목적, 평가 포인트, 필수 조건, 코워킹 질문을 자동으로 뽑습니다.

에이전트가 묻고, 사용자가 답하며 같이 씁니다

공고문 분석 결과를 바탕으로 다음 질문, 빠진 근거, 작성 전략을 정리합니다. 이 영역이 코워킹의 중심 노트입니다.

공동작성 질문 생성 현재 입력값을 보고 다음에 답해야 할 질문과 보완 근거를 다시 제안합니다.

좋은 사업기획서 사례를 모으는 지식방

선정 사례, 좋은 문장, 유사 공고, 평가표를 저장해 두면 현재 공고와 아이디어에 맞는 참고 조각을 찾아 GPT 질문에 넣습니다.

로컬 벡터 검색 브라우저 안에서 문서를 조각내고 키워드 벡터로 비교합니다. 서버로 자료를 보내지 않습니다.

오픈소스 분석과 GPT 고급판단을 연결합니다

로컬 앱은 공고문을 정리하고, GPT에는 평가자 관점의 판단과 문장 개선을 요청합니다. 답변을 다시 붙여넣으면 초안에 반영됩니다.

GPT 질문 만들기 현재 입력값을 바탕으로 GPT-5.5 같은 강한 모델에 던질 공동기획 질문을 생성합니다.
오픈소스로 강화 가능한 영역 문서 파싱, 키워드 추출, 배점/마감/제출서류 탐지, 체크리스트, 로컬 저장은 오픈소스로 충분합니다. 평가자 관점의 해석, 아이디어 전환, 설득력 있는 문장화는 강한 GPT 모델을 연결하는 편이 빠릅니다.

좋은 문서의 구조를 우리 사업의 판단 기준으로 바꿉니다

샘플 사업계획서, 공고 해설, 유사 선정사례에서 좋은 논리와 표현을 뽑아내고 적용 방식을 정리합니다.

작성자의 아이디어를 지우지 않고 선명하게 만듭니다

현장 경험, 문제의식, 하고 싶은 일을 먼저 모은 뒤 지원사업 언어로 번역합니다.

문제를 사업의 언어로 정의합니다

불편의 증상, 구조적 원인, 기존 해결책의 한계를 분리하면 필요성과 타당성이 살아납니다.

좋은 사업계획서는 인과관계가 보입니다

투입부터 중장기성과까지 연결해 평가자가 납득할 수 있는 변화의 경로를 만듭니다.

성과지표는 사업 목적의 증거입니다

과정, 산출, 결과, 확산 지표를 구분하고 측정 방법까지 설계합니다.

구분 지표명 목표값 측정 방법

실행 가능한 계획으로 바꿉니다

추진 일정, 역할, 예산, 리스크를 설계해 계획서가 실제 운영될 수 있음을 보여줍니다.

서비스 디자인 관점으로 운영 경험을 설계합니다

사용자 여정과 운영 접점을 분리하면 실행계획의 해상도가 올라갑니다.

단계 사용자 행동 접점 백스테이지 운영 실패 가능성
