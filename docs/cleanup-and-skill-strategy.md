# AI 사무국 정리 및 스킬 활용 전략

## 기본 판단

AI 사무국 앱은 모든 기능을 직접 품는 도구가 아니라, 작업을 지휘하고 결과를 모으는 로컬 기획 지휘실로 유지한다. 전문성은 `prompts/agents`와 `prompts/specialists`에 쌓고, 민감한 자료와 원본 자동화 파일은 `private/` 또는 외부 로컬 폴더에 둔다.

## 계속 유지할 기능

- AI 사무국: 명령 입력, 작업 분해, 역할 배정, 초안 생성, 아침 보고서
- 공고 해석: 지원사업 공고문을 평가 기준과 필수 조건 중심으로 읽는 기능
- 사례 모으기: 공개 가능한 선정사례, 공고문, 평가표를 브라우저 저장소에 모아 검색
- AI 질문: 무료/로컬 기능으로 질문을 만들고, 사용자가 유료 GPT/Claude에 복사해 고도화
- 초안 조립: 사업계획서, 콘텐츠, 행정문서 초안을 Markdown으로 조립
- 백업/연결: 로컬 백업 파일과 Google Drive 보관 구조

## 스킬 또는 프롬프트로 분리할 기능

- 인스타그램 콘텐츠: 카드뉴스, 릴스, 캡션, 콘텐츠 캘린더
- 디자인 디렉션: 브랜드 톤, 카드 레이아웃, 시각자료 검수
- 개발부장: 요구사항 분해, 코드 리뷰, 자동화 설계, PR 설명 작성
- 보안 검토: 개인정보, `.env`, 토큰, 자동 발송 위험, 공개 저장소 업로드 전 점검
- 행정/메일 문체: 공문, 결과보고서, 협조 메일, 강사 섭외 메일

## 보류할 기능

- 자동 메일 발송
- SNS 자동 게시
- 공문 자동 제출
- 시크릿이 필요한 GitHub Actions 실제 워크플로우 생성
- 외부 에이전트 폴더 전체 GitHub 업로드
- SaaS 전환과 다중 사용자 권한 관리

## 정리 후보

- 중복 문서: 같은 계획이 여러 파일에 흩어지면 `docs/agent-office-prd-v2.md`와 `docs/agent-office/local-automation-architecture.md`를 기준 문서로 유지한다.
- 검증되지 않은 자동화 스크립트: 계정 로그인, 게시, 예약 발행 스크립트는 공개 저장소에 올리지 않는다.
- 원본 에이전트 페르소나: 개인 정보, 이미지 링크, 계정 정보가 섞인 원문은 요약 프롬프트로만 재구성한다.
- 임시 압축파일, 일정파일, 로그: `private/` 또는 원본 로컬 폴더에만 보관한다.

## 폴더 운영 원칙

```text
planning-copilot-master/
  src/                    앱 실행 코드
  prompts/agents/          AI 사무국 기본 역할 프롬프트
  prompts/specialists/     외부 전문 에이전트 지식을 정제한 프롬프트
  docs/                    설계, 운영, 자동화 문서
  automation/              예시 설정과 로컬 자동화 안내
  private/                 민감 자료와 원본 요약, Git 업로드 금지
```

## 작업 흐름

1. 앱에서 명령을 입력해 사무국 초안과 GPT 질문 패키지를 만든다.
2. 민감정보를 제거한 뒤 통합 질문을 유료 GPT/Claude에 복사한다.
3. 답변이 부족하면 `prompts/specialists`의 전문 프롬프트를 추가로 붙인다.
4. 검토된 답변만 앱의 초안 조립, 보고서, 콘텐츠 계획에 반영한다.
5. 공개 가능한 산출물만 GitHub에 커밋하고, 원본 문서와 민감정보는 로컬/Drive에 둔다.

## 판단 기준

- 반복해서 쓰는 지식은 프롬프트로 만든다.
- 계정, 토큰, 개인정보가 닿는 자동화는 기본적으로 로컬 전용으로 둔다.
- 사용자가 승인해야 하는 일은 버튼 하나로 자동 실행하지 않는다.
- 공개 저장소에는 재사용 가능한 구조, 문서, 예시만 남긴다.
